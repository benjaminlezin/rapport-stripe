📦 Souscription

Le client peut s'abonner directement depuis l'application Abby aux plans :

Start mensuel Start annuel Pro mensuel Pro annuel Business mensuel Business annuel
⬆️ Upgrade in-app

Le client peut monter en gamme seul depuis son espace :

Basique → Start / Pro / Business Start → Pro / Business Pro → Business Mensuel → Annuel
🎫
Souscription & upgrade — Pourquoi le support intervient

Même si ces actions sont disponibles en autonomie, le support les prend en charge quand le client a rencontré un bug lié à un code promo mal paramétré (ex : code à 1 € au lieu de 20 %). Plutôt que de risquer un churn, on envoie directement une facture par email avec la réduction correcte appliquée.

🔴
Churn & annulation — Pourquoi le support intervient

Deux situations nécessitent une intervention manuelle :

  • Geste commercial — passage manuel d'une facture en statut "Non recouvrable" dans Stripe (ex : impayé justifié, compensation client).
  • Cessation d'activité — annulation immédiate avec remboursement manuel au prorata des jours non consommés.
📦 Gestion des abonnements
1
Moyen Souscription avec code promo via lien de paiement ~5 min ▼

🎯 Contexte

Créer un abonnement Stripe pour un client en appliquant un code promo, et lui envoyer un lien de paiement sécurisé. À utiliser uniquement avec accord écrit du client.

✅ Prérequis

  • Accord écrit du client (email ou Intercom)
  • Plan souhaité identifié (Start / Pro / Business)
  • Durée confirmée (mensuel / annuel)
  • Code promo valide disponible

📋 Procédure step-by-step

1
Stripe Dashboard → rechercher le client par email dans la barre de recherche
2
Ouvrir la fiche client → onglet Abonnements → cliquer + Créer un abonnement
3
Configurer le plan : sélectionner le produit Abby + la variante (Start/Pro/Business) + la durée (mensuel/annuel)
4
Cliquer sur Ajouter un coupon → saisir le code promo → vérifier que la réduction s'applique correctement
5
Dans la section Mode de paiement → sélectionner Envoyer un lien de paiement
⚠️ Ne JAMAIS sélectionner "Débiter automatiquement" sans consentement écrit explicite pour la CB du client
6
Vérifier le récapitulatif : plan, réduction, montant final, date de facturation
7
Cliquer Créer l'abonnement → le lien de paiement est envoyé automatiquement par email au client
8
Ajouter une note dans Activité récente du client : préciser le plan, le code promo appliqué, et la source de l'accord (ex: "Abo Pro annuel créé - code promo LANCEMENT20 appliqué - accord email du 12/06")
🕐 ~5 min
👤 Agent

Élevé Upgrade / Upsell d'abonnement (24 combinaisons) ~8 min ▼

🎯 Contexte

Faire monter en gamme un abonnement existant (ex: Start → Pro, mensuel → annuel). L'upgrade est immédiat avec prorata. Valable pour 24 combinaisons possibles de plans × durées.

✅ Prérequis

  • Accord écrit du client (email ou Intercom)
  • Abonnement en état Actif ou En retard
  • Plan de destination identifié

⚠️ Vérification préalable critique

!
Si l'abonnement est en état "Annulation prévue" (fin de période en cours) : aller dans les 3 points ··· de l'abonnement → Ne pas annuler → confirmer. L'abonnement redevient Actif. Ensuite procéder à l'upgrade.
⚠️ Sans cette étape, l'upgrade est bloqué

📋 Procédure step-by-step

1
Stripe → fiche client → onglet Abonnements → ouvrir l'abonnement actif
2
Cliquer Actions (haut droite) → Mettre à jour l'abonnement
3
Dans la liste des postes : cliquer Ajouter un poste → sélectionner le nouveau produit (plan de destination)
4
Cliquer sur l'ancien poste → Supprimer (icône corbeille)
5
Ouvrir l'onglet Facture → vérifier le montant prorata généré
💡 L'upgrade génère une facture prorata immédiate. Vérifier que le calcul est correct avant de valider.
6
Cliquer Mettre à jour l'abonnement → confirmer
7
Ajouter une note dans Activité récente : préciser l'ancien plan, le nouveau plan, la date, et la source de l'accord
🕐 ~8 min
👤 Agent
⚡ Complexité Élevée
💰 Prorata immédiat
📒 Voir process Notion
3
Élevé Downgrade d'abonnement ~8 min ▼

🎯 Contexte

Faire descendre un abonnement vers un plan inférieur (ex: Pro → Start). Contrairement à l'upgrade, le downgrade prend effet en fin de période — pas de remboursement prorata.

✅ Prérequis

  • Accord écrit du client
  • Abonnement en état Actif
  • Identifier la prochaine date de facturation

📋 Procédure step-by-step

1
Stripe → fiche client → Abonnements → ouvrir l'abonnement actif
2
Actions → Mettre à jour l'abonnement
3
Ajouter le nouveau plan (plan inférieur) → Supprimer l'ancien plan
4
Dans les options de mise à jour, sélectionner En fin de période actuelle
⚠️ NE PAS choisir "Immédiatement" pour un downgrade — cela facturerait le client deux fois
5
Valider → L'abonnement affiche le statut "Annulation prévue" avec le nouveau plan effectif à partir de la prochaine date de facturation
6
Envoyer la macro "Confirmation downgrade" au client dans Intercom
7
Ajouter une note dans Activité récente : ancien plan → nouveau plan, date effective

📅 Règles de timing

  • Abonnement mensuel → downgrade effectif en fin du mois en cours
  • Abonnement annuel → downgrade effectif en fin de l'année en cours
  • Aucun remboursement prorata dans les deux cas
🕐 ~8 min
👤 Agent
⚡ Complexité Élevée
📅 Effectif fin de période
📒 Voir process Notion

🎯 Contexte

✅ Prérequis

  • Accord écrit du client
  • Abonnement Actif
  • Clarifier le sens du changement (upgrade de durée vs downgrade)

📋 Procédure step-by-step

1
Stripe → fiche client → Abonnements → ouvrir l'abonnement actif
2
Actions → Mettre à jour l'abonnement
3
Cliquer sur le plan actuel → le remplacer par la variante de même niveau mais période différente (ex: "Abby Pro Mensuel" → "Abby Pro Annuel")
4
Choisir le timing :
  • Mensuel → Annuel : Immédiatement (upgrade = prorata)
  • Annuel → Mensuel : En fin de période (downgrade)
5
Vérifier l'onglet Facture (prorata si immédiat) → valider
6
Ajouter une note dans Activité récente

6
Critique Annuler un abonnement (3 sous-cas) ~6 min ▼

🎯 Contexte

Annuler définitivement un abonnement Stripe. La procédure varie selon le cas : fin d'essai, fin de période normal, ou cessation d'activité avec remboursement.

✅ Prérequis

  • Identifier le cas parmi les 3 sous-cas
  • Vérifier les critères d'éligibilité au remboursement si applicable
  • Accès au raccourci Stripe depuis Intercom

📋 Procédure commune (étapes 1-4)

1
Depuis Intercom : utiliser le raccourci Stripe pour ouvrir directement la fiche du client
2
Dans la fiche client Stripe → section Abonnements → cliquer sur l'email/nom de l'abonnement actif
3
Cliquer Actions (haut droite) → Annuler l'abonnement
4
Choisir le mode d'annulation selon le sous-cas (voir ci-dessous)
Cas 1 — Fin d'essai 14 jours (pas de paiement effectué)

Sélectionner Immédiatement → aucun remboursement car aucun paiement n'a eu lieu. Confirmer l'annulation.

Cas 2 — Annulation en fin de période (client ne veut plus renouveler)

Sélectionner En fin de la période actuelle → l'accès reste actif jusqu'à la prochaine date de facturation, puis l'abonnement s'annule. Pas de remboursement prorata.

Cas 3 — Cessation d'activité (remboursement prorata)

Sélectionner Immédiatement → cocher "Rembourser le montant au prorata" → confirmer. Le remboursement est calculé sur les jours non utilisés.

⚠️ Vérifier l'éligibilité au remboursement avant de procéder (justificatif de cessation si nécessaire)
🕐 ~6 min
👤 Agent
⚡ Complexité Critique
📒 Voir process Notion

Moyen Mise en pause de l'abonnement ~6 min ▼

🎯 Contexte

Suspendre temporairement un abonnement pendant 3 mois maximum. Limité à 1 fois par an par client. Sert à éviter un churn en proposant une alternative à l'annulation.

✅ Prérequis

  • Vérifier la balise bot:pauseabochurn dans Intercom (= déjà fait cette année)
  • Maximum 1 pause par an
  • Abonnement en état Actif

📋 Procédure step-by-step

1
Vérifier dans Intercom si la balise bot:pauseabochurn est présente sur le contact
⚠️ Si la balise est présente → envoyer la macro "Mise en pause impossible" et clore la conversation
2
Stripe → fiche client → ouvrir l'abonnement actif
3
Actions → Suspendre le renouvellement
4
Calculer la date de reprise : date_prochaine_facture + 3 mois
💡 Exemple : prochaine facture le 15 juillet → reprise le 15 octobre
5
Saisir la date de fin de pause calculée → confirmer la suspension
6
Envoyer la macro "Mise en pause effectuée" au client dans Intercom
7
Ajouter la balise bot:pauseabochurn dans Intercom sur le contact
🕐 ~6 min
👤 Agent
⚡ Complexité Moyenne
📅 Max 1×/an
📒 Voir process Notion

Critique Remboursement client ~10 min ▼

🎯 Contexte

Rembourser tout ou partie d'une somme déjà prélevée. À utiliser uniquement dans des cas justifiés (cessation d'activité, erreur de facturation). La préférence est toujours au crédit Stripe.

⚡ Règles importantes

  • Préférer le crédit Stripe à un remboursement réel (plus simple, pas de délai bancaire)
  • Remboursement réel uniquement si litige → escalader au superviseur
  • Ne jamais rembourser sans validation de la raison

📋 Option A — Remboursement via annulation immédiate avec prorata

1
Stripe → abonnement → Actions → Annuler l'abonnement
2
Sélectionner Immédiatement → cocher "Rembourser le montant au prorata"
3
Vérifier le montant calculé → confirmer

📋 Option B — Remboursement direct depuis une transaction

1
Stripe → fiche client → onglet Paiements → ouvrir le paiement concerné
2
Cliquer Rembourser le paiement (bouton haut droite)
3
Saisir le montant (partiel ou total) → ajouter une raison → confirmer
4
Ajouter une note dans Activité récente avec la justification du remboursement
⚠️ En cas de litige → NE PAS procéder seul. Escalader au superviseur avant tout remboursement.
🕐 ~10 min
👤 Agent + superviseur si litige
⚡ Complexité Critique
8
Critique Passage d'une facture en "Non Recouvrable" ~5 min ▼

🎯 Contexte

Marquer une facture comme irrécupérable pour éviter que Stripe ne relance des tentatives de prélèvement automatiques, et pour débloquer l'accès à l'app si nécessaire.

⚡ Pourquoi c'est critique

  • Sans ce statut : Stripe retente le prélèvement automatiquement (jusqu'à 4 fois)
  • Peut bloquer l'accès à l'application Abby
  • Facture reste "En retard" indéfiniment sans action

📋 Procédure step-by-step

1
Stripe → fiche client → onglet Factures → ouvrir la facture concernée
2
Cliquer sur les 3 points ··· en haut à droite de la facture
3
Sélectionner "Modifier l'état de la facture"
4
Dans la liste déroulante, choisir Non recouvrable
5
Ajouter une note expliquant la raison (ex: "Client en cessation d'activité - accord superviseur")
6
Cliquer "Mettre à jour l'état" → vérifier que la facture affiche bien "Non recouvrable"
✅ La facture ne génèrera plus de tentatives de prélèvement automatiques
🕐 ~5 min
👤 Agent
⚡ Complexité Critique
📒 Voir process Notion

Élevé Gestion des taux de TVA réduits (DROM — 8,5%) ~15 min ▼

🎯 Contexte

Pour les clients situés dans les DOM-ROM (Guadeloupe, Martinique, Réunion, Guyane, Mayotte), la TVA est de 8,5% au lieu de 20%. Nécessite une correction manuelle via un workflow à 2 niveaux (Agent → Team Lead).

✅ Prérequis

  • Vérifier l'adresse du client (DOM-ROM confirmé)
  • Identifier l'état de la facture (Payée ou En attente)
  • Escalader au Team Lead — l'agent ne procède pas seul

📋 Étape 1 — Agent (vérification + escalade)

1
Vérifier l'adresse du client dans Abby → confirmer DOM-ROM (éligible TVA 8,5%)
2
Créer un ticket interne pour le Team Lead avec : nom client, ID Stripe, état de la facture, montant
Cas A — Facture déjà Payée (correction rétroactive)
1
Stripe → ouvrir la facture payée → Annuler la facture
2
Rembourser via avoir (créer une note de crédit)
3
Créer une nouvelle facture avec TVA 8,5% → valider via un crédit Stripe de 0€ pour déclencher la validation
4
Mettre à jour l'abonnement avec le taux réduit pour les prochaines factures
Cas B — Facture En attente (pas encore prélevée)
1
Stripe → annuler la facture en attente
2
Recréer une nouvelle facture avec TVA 8,5% → envoyer le lien de paiement
🕐 ~15 min
👥 Agent + Team Lead
⚡ Complexité Élevée
🇷🇪 DROM 8,5%

Élevé Conversion abonnement annuel → multi-années (2 ou 3 ans) ~15 min ▼

🎯 Contexte

Permettre à un client de s'engager sur 2 ou 3 ans avec un paiement échelonné (2 ou 3 fois). Implique la création de plusieurs factures planifiées manuellement dans Stripe.

✅ Prérequis

  • Convenir de la durée (2 ou 3 ans) avec le client
  • Convenir de l'échelonnement (2 ou 3 fois)
  • Maximum 3 versements

📋 Procédure step-by-step (exemple : 2 ans en 2 fois)

1
Calculer le montant total : tarif annuel × nombre d'années
💡 Exemple : Abby Pro 2 ans = 216€ × 2 = 432€ en 2 versements de 216€
2
Stripe → fiche client → Solde créditeur → Réajuster le solde → ajouter un crédit égal au montant total de l'abonnement
3
Valider l'abonnement via le crédit (la prochaine facture sera absorbée par le crédit)
4
Créer la Facture 1 (débit immédiat) :
  • Description : "Abby Pro 1/2 — Abonnement 2 ans"
  • Montant : 1er versement
  • Mode : Demander un paiement
5
Créer la Facture 2 (planifiée à M+1 ou M+12) :
  • Description : "Abby Pro 2/2 — Abonnement 2 ans"
  • Montant : 2e versement
  • Mode : Planifier la date d'envoi → date = mois suivant
6
Si 3 fois : créer une Facture 3 planifiée à M+2 (ou M+24) → description "Abby Pro 3/3"
7
Ajouter une note dans Activité récente : récapituler le plan, la durée, les montants et les dates de prélèvement
🕐 ~15 min
👤 Agent
⚡ Complexité Élevée
📄 Max 3 versements
📒 Voir process Notion

Élevé Suppression du compte Stripe Express lié à Abby ~8 min ▼

🎯 Contexte

Supprimer le compte Stripe Express d'un client (compte utilisé pour percevoir des paiements via Abby). Irréversible — nécessite des prérequis stricts.

⚠️ Prérequis stricts

  • Solde du compte = 0,00 €
  • Aucun virement en cours ou en attente
  • Vérifier dans le dashboard Stripe Express

📋 Procédure step-by-step

1
Stripe Dashboard → fiche client → faire défiler vers le bas jusqu'à la section Stripe Express
2
Vérifier le solde disponible : doit être 0,00 €
⚠️ Si le solde est > 0 : attendre le virement automatique ou déclencher un paiement manuel avant de supprimer
3
Vérifier qu'aucun virement n'est en cours (statut "En cours" dans la section virements)
4
Cliquer sur les 3 points ··· à côté du compte Stripe Express → Supprimer
5
Confirmer la suppression dans la popup de confirmation
6
Envoyer la macro de confirmation de suppression au client dans Intercom
🕐 ~8 min
